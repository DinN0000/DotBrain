# DotBrain UX Improvement Design

**Date**: 2026-02-15
**Goal**: 사내 배포를 위한 전반적 UX 개선. PKM 초보자가 파일 정리 자동화를 바로 활용할 수 있도록.

**핵심 원칙**: 사용자가 구조를 결정하고(프로젝트명, 폴더), AI가 그 안에서 분류를 실행한다.

---

## 1. 온보딩 재설계

현재 4단계 → 5단계로 확장. 텍스트, 시각적 가이드, 동기부여 모두 개선.

### Step 1 — "왜 DotBrain인가" (동기부여)
- Before/After 비교 시각화
  - Before: 바탕화면에 `회의록_최종_진짜최종.pdf`, `보고서(2).docx` 등 산재
  - After: 프로젝트별로 깔끔하게 정리된 폴더 구조
- "파일을 던지면, AI가 알아서 정리합니다" 한 문장 핵심 가치

### Step 2 — "내 공간 만들기" (폴더 설정)
- PKM 루트 경로 선택
- 선택 즉시 폴더 구조 라이브 프리뷰 트리 표시
- PARA 각 폴더를 일상 비유로 설명:
  - Project — 책상 위 (진행 중인 일, 마감 있음)
  - Area — 서랍 (늘 관리하는 것)
  - Resource — 책장 (참고 자료)
  - Archive — 창고 (끝난 것)

### Step 3 — "프로젝트 등록" (사용자가 구조 결정)
- "지금 진행 중인 일에 이름을 붙여주세요" 안내
- 예시 제공: "2026 마케팅 캠페인", "신규 서비스 런칭"
- 핵심 안내: "AI는 여기 등록된 프로젝트 안에서만 파일을 분류합니다. 새 프로젝트가 필요하면 언제든 추가할 수 있습니다"
- 최소 1개 필수, "나중에 추가" 옵션 명시

### Step 4 — "AI 연결" (API 키)
- "AI가 파일을 읽고 분류합니다. API 키가 필요합니다" 설명
- API 키 발급 링크 직접 제공
- 키 입력 후 즉시 연결 테스트 + 성공/실패 피드백
- 안내 추가: "API 키 없이도, 만들어진 폴더에 Claude Code를 연결해서 사용할 수 있습니다"

### Step 5 — "첫 파일 정리해보기" (체험)
- 온보딩 마지막에 샘플 파일 1~2개로 실제 분류 체험
- 즉시 결과를 보여줘서 학습 완료

## 2. 인박스 → 분류 → 결과 흐름 개선

### 인박스 화면
- 파일 드롭 후 미리보기 썸네일 (PDF 첫 페이지, 이미지 축소판, 텍스트 첫 줄)
- 파일 개수 + 예상 소요 시간 ("3개 파일, 약 10초")
- "정리하기" 버튼 더 강조

### 처리 중 화면
- 상단: 전체 진행률 바 + "37/100 완료" 카운터
- 하단: 현재 처리 중인 파일 1~2개만 이름 표시
- 완료된 파일은 카운트만 (100개도 성능 문제 없음)
- 기존 배치 처리 구조(batchSize 10, 동시 3) 유지

### 결과 화면
- 전체 목록은 LazyVStack으로 (대량 파일 대응)
- 전문 용어 제거:
  - "confidence: 0.92" → 표시 안 함 (낮을 때만 "확인 필요")
  - "relocated" → "더 적합한 위치로 옮겨짐"
  - "frontmatter injected" → 표시 안 함
- 확인 필요 항목: "이 파일이 어디에 들어갈지 모르겠어요. 골라주세요" + 선택지 버튼
- 프로젝트 미매칭: "등록된 프로젝트에 맞는 곳이 없습니다. 새 프로젝트를 만드시겠습니까?" (AI가 임의 생성하지 않음)

## 3. 대시보드 & 네비게이션 개선

### 화면 구조 — 3계층

```
메인 (하단 탭)
├── 인박스        ← 핵심 행동
├── 대시보드      ← 볼트 현황 + 관리 진입점
└── 설정          ← API 키, 폴더 경로

대시보드 하위
├── PARA 관리     ← 폴더 이동/생성
├── 프로젝트 관리  ← 프로젝트 추가/아카이브
├── 검색          ← 볼트 검색
└── 볼트 관리     ← 오류 검사 + 태그 보완 + MOC 갱신 + 전체 재정리 통합
```

### 핵심 변경
- 대시보드: 통계 카드 + 하위 기능 진입 카드 (직접 실행이 아닌 허브 역할)
- "볼트 관리" 통합: 흩어진 유지보수 기능을 하나로
- 하단 푸터: 인박스 / 대시보드 / 설정 3개만
- 검색은 대시보드 하위로, 도움말은 설정 안으로

### 네비게이션
- 모든 하위 화면에 breadcrumb: `대시보드 > PARA 관리`
- 뒤로가기는 항상 부모 화면으로

## 4. 용어 & 마이크로카피 개선

### 원칙
- PARA는 핵심 체계이므로 온보딩에서 잘 설명하고 앱 전반에서 자연스럽게 사용
- 내부 구현 용어만 숨김 (confidence, frontmatter, MOC, relocated 등)
- 에러 메시지는 "~할 수 없습니다" 대신 "~해주세요" (행동 유도형)

### 용어 매핑

| 내부 용어 | 사용자에게 보이는 표현 |
|-----------|----------------------|
| confidence: 0.85 | (표시 안 함, 낮을 때만 "확인 필요") |
| relocated | 더 적합한 위치로 옮겨짐 |
| frontmatter injected | (표시 안 함) |
| MOC 갱신 | 폴더 요약 업데이트 |
| NoteEnricher | 태그·요약 자동 보완 |
| VaultAuditor | 오류 검사 |
| PKM Root Path | 내 지식 폴더 |
| 프로젝트 미매칭 | 이 파일이 어디에 들어갈지 모르겠어요 |
| Project / Area / Resource / Archive | 그대로 사용 (온보딩에서 학습) |

---

## 제약사항

- AI는 사용자가 등록한 프로젝트 안에서만 분류. 새 프로젝트를 임의 생성하지 않음.
- 프로젝트 미매칭 시 사용자에게 새 프로젝트 생성 여부를 묻는다.
- API 키는 개인별 설정 방식.
- 메뉴바 팝오버 크기 제한 내에서 동작해야 함.
